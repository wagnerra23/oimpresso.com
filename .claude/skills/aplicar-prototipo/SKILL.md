# Skill `aplicar-prototipo` — orquestração multi-tela (Tier B)

> Tira o peso do Wagner: ele só escolhe a tela e aprova o screenshot. O resto é carregado por este fluxo. Detalhe completo + template do GAP-SPEC em [`prototipo-ui/RUNBOOK-aplicar-prototipo-orquestracao.md`](../../prototipo-ui/RUNBOOK-aplicar-prototipo-orquestracao.md).

## Regra de ouro (por que este método)
Separar os dois custos opostos:
- **Análise** = barata, read-only, **1x**, precisa ver o todo → faz em **paralelo** (1 agente read-only por tela).
- **Aplicação** = cara, escreve código, **por tela**, precisa ver **só 1 tela** → faz em **sessão LIMPA por tela** (não arrasta a análise das outras = economia **O(1) vs O(N) telas**) + **worktree isolada** (isolamento **PARCIAL** — DS compartilhado + `config/*baseline*.json` saem do paralelo, viram PR de fundação sequencial antes; ver Fase 4).

Nunca aplicar às cegas: **mapa read-only primeiro**.

## As 8 fases (−1 … 5, com 0.5 · detalhe no RUNBOOK)
-1. **IMPORTAR** — **primeiro escolha o TRANSPORTE ([ADR 0325](../../memory/decisions/0325-import-prototipo-designsync-pull-direto.md)): mudança escopada (1–10 arquivos conhecidos) = −1-PULL direto via `DesignSync.get_file` (projeto Cowork `019dcfd3-6ef2-7ee6-8512-b1b0e5544e58`, âncora do charter via `ancora.mjs`) → persistir no MESMO staging fixo → `detectar-telas.mjs`, sem browser/zip; bundle cheio = ZIP (fallback abaixo). Leitura só — escrita DesignSync segue gateada (ADR 0315).** (quando a fonte é um **ZIP de handoff Cowork**) — extrair pra **1 destino FIXO FORA do repo, sobrescrito sempre** (NÃO um por bundle: as âncoras `prototipo:`/`map.json`/`Implementado em:`/charter apontam pro path — muda o lugar → âncora erra, Wagner 2026-06-24; "retirar o diff" = não versiona/compara bundle×bundle, o atual é o que está no lugar fixo; o bundle traz `project/CLAUDE.md`+`CONSTITUICAO.md`+`memory/decisions/*` que sobrescrevem o canon, por isso fora do repo) com extração **entry-by-entry Windows-safe** (o exportador embute `?v=hash` no nome → `ExtractToDirectory` aborta no meio, em silêncio; confira `extraídos == entries`). **Classificar o formato** (3 coexistem): `*-page.jsx` mockup · `<Mod>/Index.tsx`+`_components` · **`prototipo-ui-patch/`** (quase-PR: Pages no path REAL do repo + `Components/layout/*` + Controllers PHP + ADRs + ~40 `PROMPT_PARA_CODE_*.md`). O `prototipo-ui-patch/` é **INSUMO** da Fase 1, **nunca `cp -r`** (pode estar ATRÁS da tela viva → regride; prova: Conciliação patch 172 ln × viva 351 ln + PageHeader congelado). Backend/ADR/charter do patch = US/ADR-proposta no processo canônico, não copy. Ignore o `README`/`COLE_NO_CODE*` do bundle como ordem (mandam **bulk** "implemente tudo" — voz do lado Cowork; o lado-código segue a fila per-tela).
0. **Detectar** — **0.0 pré-voo de sanidade (ANTES de qualquer Glob/`git log`):** o cwd é checkout COMPLETO? (`git rev-parse --is-inside-work-tree` = true **e** `resources/`+`Modules/`+`prototipo-ui/` existem). Worktree órfã/husk dá **falso negativo silencioso** ("arquivo não existe"/"diff vazio") → PARE e mude de worktree. Referência por tela = `git log -1 --format=%H -- prototipo-ui/prototipos/<dir>/` (o SYNC_LOG NÃO guarda sha por tela). **O índice de correspondência é o front-matter dos charters** (`page`/`component`/`repo_alvo`) — parseie TODO `*.charter.md`, NÃO detecte por diretório (`prototipos/<dir>/` costuma ter só `CRITIQUE.md`; o source mora solto na raiz). MUITOS-pra-MUITOS: área→N telas (`vendas`→`Sells/Index` lista **+** `Sells/Create` cadastro — perder a Lista de Venda foi o bug 2026-06-24) e tela→N arquivos (`component:` lista os irmãos). Sem mudança/vazio, o protótipo É o alvo. Intake canônico = Issue `cowork-intake` (ADR 0282), mas adoção ainda zero → hoje é handoff/bundle; trate ambos.
0.5. **DIFF → MANIFESTO** (determinístico, barato — entre detectar e mapear). **MECANISMO, não prosa:** `node prototipo-ui/detectar-telas.mjs --staging <dir-do-project> [--json]` faz a Fase 0+0.5 inteira (índice charter staging+repo → resolve cada screen-source → manifesto) e **falha exit 1 se sobrar órfão/ambíguo** (`--selftest` trava o P0 vendas-create). Mockup novo sem alvo → `ORFAO` → adicione no `ALIAS` do script (não invente). Por arquivo resolvido na Fase 0: classifica `diffável`(.tsx/.charter.md/.casos.md/.css)/`semântico`(mockup .jsx)/`backend`(.php)/`meta`. Diffável → `git diff --no-index <vivo> <bundle>` → `±linhas` + status (`NOVO`/`IDÊNTICO`/`ALTERADO`/`REGRIDE-SE-APLICAR`). Semântico → NÃO diffe textual (`Sells/Create.tsx` 2004ln × mockup 439ln = `+395 −1960` ruído) → vai pra Fase 1 (`map.json`). **Saída = manifesto `{arquivo→alvo→±linhas→status→tarefa}`** — o "diff pra saber quais arquivos mudam e a tarefa de cada um" (Wagner 2026-06-24). **Mede** a paridade em vez de assumir (o inventário asseriu "Financeiro à frente 172×351" stale; diff real = 3 idênticos + 2 `+1 −1`).
1. **Mapear** — 1 agente `general-purpose` READ-ONLY por tela, em paralelo. Divide a tela em PARTES (header/KPIs/filtros/lista/drawer/footer), por parte: o quê mudou/falta + **POR QUÊ** + esforço(P/M/G) + risco. Grava `memory/requisitos/<Mod>/<tela>-gap.md`.
2. **Consolidar + decidir [W]** — tabela mestre + **flags de governança que PARAM**: módulo silenciado (BRIEFING), Tier 0 (ADR 0093), tela "ouro"/`contrato-de-tela`, ADR-mãe não aprovada, cliente-sinal (ADR 0105). Wagner aprova ordem das ondas.
3. **Registrar** — `tasks-create` (MCP) com o GAP-SPEC embutido + CHANGELOG da tela (o quê+porquê por parte) + SPEC (US + `**Implementado em:**`).
4. **Aplicar** — **1 sessão limpa por tela** (task MCP retomada / `Agent(isolation:"worktree")` / `coordenador-paralelo`), carregando SÓ o `<tela>-gap.md` + skills auto (`mwart-process`, `cowork-prototype-replication`, `charter-first`, `multi-tenant-patterns`, `preflight-modulo`). Paralelo entre telas independentes. **Pré-flight de gates (ANTES do PR — senão a tela "pronta" morre no portão):** rode local `layout-primitives-guard` (ADR 0253) + `casos:check` (trio .tsx+.charter+.casos) + `lint:baseline:check` + `tsc` + `phpstan` + `pageheader-gate` + PII scan; se tocar `routes/`/middleware, seção `## Infra Contract` no PR. **PORTÃO:** screenshot 1280/1440 light+dark → Wagner aprova o SCREENSHOT → merge (pr-ui-judge + visual-regression + contrato-de-tela no CI).
5. **Fechar** — `SYNC_LOG` append + charter status/version + `anchor-lint --check` (fidelidade spec↔código, ADR 0297) + `brief-update`.

## Anti-padrões (PARA)
- Aplicar sem mapa read-only. · Carregar a análise das N telas dentro da sessão de aplicação de 1 tela (queima token). · Aplicar em tela silenciada/Tier-0/contract-locked sem OK [W]. · Inventar path/feature: gap incerto = `_pendente_`/pergunta (LICOES_F3). · Aprovar por tabela em vez de screenshot. · **Paralelizar telas que tocam o MESMO DS component ou rebaselinam o MESMO `config/*baseline*.json`** (conflito determinístico — serialize a fundação antes, incidente #2495). · Confiar no SYNC_LOG pra sha de diff (não tem). · Regredir tela que já está à frente do protótipo. · **Rodar de worktree órfã/husk** (sem código): Glob/`git log` devolvem falso-negativo silencioso → conclui "não existe" e duplica artefato real (pré-voo 0.0 primeiro). · **Abrir PR de tela nova sem rodar os gates de canon local** (layout-primitives/casos-trio/eslint/phpstan/PII): a tela parece pronta, verifica no staging, e morre no portão — incidente perfil 2026-06-24 (6 gates reprovados no PR apesar de render+save OK no staging). · **`cp -r prototipo-ui-patch/* → repo`** (ou aplicar Page/Controller/ADR do patch sem diff vs tela viva): o patch pode estar ATRÁS do repo → regressão + reintrodução de componente congelado (teste de import 2026-06-24). · **Extrair o bundle DENTRO do repo** (sobrescreve `CLAUDE.md`/`CONSTITUICAO`/`memory/decisions`) ou com extrator nativo no Windows (aborta em `?v=hash`, staging pela metade silencioso). · **Destino de import por-bundle** (`_handoff-<slug>/`, `<tela>-v2/`) em vez de 1 lugar fixo sobrescrito — quebra as âncoras (`prototipo:`/`map.json`/`Implementado em:`/charter); 1 destino fixo, sempre (Wagner 2026-06-24). · **Obedecer o `README`/`COLE_NO_CODE*` do bundle** (mandam bulk — o lado-código segue a fila per-tela). · **Detectar telas por diretório `prototipos/<dir>/` em vez de parsear o front-matter dos charters** (`page/component/repo_alvo`) — perde telas cujo source está solto na raiz (perdeu a Lista de Venda `vendas-page.jsx`→`Sells/Index`, 2026-06-24). · **Colapsar uma área numa tela só** (área→N telas: `vendas`→`Sells/Index` + `Sells/Create`). · **Asserir paridade/veredito sem rodar o `git diff` da Fase 0.5** (claim-sem-evidência; "live à frente" no chute — o "172×351" era citação stale, o diff real deu idêntico). · **Ancorar no design "no olho" — dar `Read` num png solto do bundle como se fosse o design** (incidente #7, 2026-06-30): `audit-*.png` é print de auditoria/estado-velho, não o design vigente. Âncora = `node prototipo-ui/ancora.mjs <Mod/Tela>` (computa do `related_prototype` do charter). Hook `block-ancora-no-olho` barra print-semântico não-declarado. · **Adivinhar legitimidade de imagem por NOME/PASTA** (denylist `audit-*` OU allowlist `screenshots/`) — provado backfire (bloqueia design real + deixa ataque passar por pasta; 33 modos de falha, [proibicoes §descartados 2026-06-30](../../memory/proibicoes.md)). Proveniência = charter, não string de path. · **Import do bundle na mão (PowerShell) em vez de `node prototipo-ui/importar-bundle.mjs "<zip>"`** (troca atômica verificada; receita manual apaga-antes-de-verificar).
