08

Voz & tom

Como falar com o usuário em microcopy, vazio, erros, sucesso

Direto, em pt-BR coloquial

Frases curtas, verbo no imperativo. Sem jargão técnico. Sem "por favor", "atenciosamente", "estamos cientes".

Bom Marque como pago quando o cliente confirmar.
Evitar Por favor, atualize o status para "Pago" quando o pagamento for confirmado pelo cliente.

Empty states com ação

Estado vazio nunca é só "Nada aqui." Sempre mostre o que fazer agora.

Bom Nenhuma OS aqui ainda. + Criar a primeira
Evitar Lista vazia.

Erros = caminho de saída

Erro sempre vem com 1 ação clara de recuperação. Nunca só "Algo deu errado."

Bom Não conseguimos salvar. Verifique a conexão e tente de novo.
Evitar Erro 500. Algo deu errado.

Números com unidade

Sempre R$, %, dias, peças — nunca número solto sem contexto.

Bom R$ [redacted Tier 0] · 7 dias · 1.200 cartões · 22%
Evitar 13420,8 · 7 · 1200 · 0.22
09

Do's & Don'ts

Regras de uso visual e de informação

✓ Fazer

  • Usar IBM Plex Mono para todo número, código, CNPJ, NFe, prazo
  • Usar a stone-900 para hero cards de cockpit (saldo, total, KPI principal)
  • Usar radius 10px em cards e 6-7px em botões/inputs
  • Manter border 1px stone-200 em todo container de superfície
  • Adicionar border-bottom dashed entre seções dentro do mesmo card
  • Tabular nums em qualquer coluna numérica de tabela
  • Status badges com bolinha colorida (não emoji)

✗ Evitar

  • Misturar fontes além de Plex Sans + Plex Mono (sem Inter, Roboto, sem Material)
  • Usar gradient colorido em hero cards — só stone-900 chapado ou gradient sutil
  • Adicionar emoji decorativo (📊 📈 💰) em UI — usar SVG ou texto
  • Usar cores fora da paleta — qualquer roxo/rosa/turquesa solto está errado
  • Texto em UPPERCASE exceto eyebrow + label (sempre mono)
  • KPIs com número grande sem contexto (faltar sub-line ou delta)
  • Mais de 3 níveis de hierarquia visual numa mesma tela
