Resolvidos vs v3

  • ✔ Geometria grid 3-col — C centraliza de fato
  • ✔ Tokens reais (--bg/--surface/--border/--text)
  • ✔ Density compact/default/comfy (não cozy)
  • ✔ Hue via data-group map em --origin-*
  • ✔ color-mix(in oklch, …) — sem HSL legado
  • ✔ --z-page-h em escala documentada
  • ✔ Subtítulo com min-2l sem pulo
  • ✔ Badge contador adota estilo já existente (.os-tab-n)
  • ✔ Glow do underline via box-shadow + color-mix

Atalhos visuais nas cards

  • · Underline ativa "morde" baseline via margin-bottom:-1px
  • · Glow = box-shadow: 0 0 8px color-mix(…hue 50% transparent)
  • · Badge ativo = bg color-mix(hue 18% bg) + fg hue
  • · kbd ⌘N dentro do primary, mono 10.5px, bg white/18
  • · Subtítulo danger com data-tone="danger" · oklch (0.55 0.18 25)
  • · Backdrop blur 8px sat 120% visível ao trocar pra scrolled

O que esse protótipo NÃO faz (escopo)

  • · Hover-prefetch (Inertia router.prefetch — só em produção)
  • · Dirty-state guard (precisa do ConfirmDialog real do projeto)
  • · Skeleton aparecer só após 100ms (mocado, mostra sempre quando ativado)
  • · i18n EN/PT toggle (PT hardcoded aqui)
  • · Underline com framer-motion spring (CSS-only neste preview; spec diz spring 380/30 em React)
  • · RTL (CSS escrito mas não exibido)
