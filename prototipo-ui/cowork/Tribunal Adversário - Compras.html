Auto Adversário da Tela /compras

O Réu é a tela. Presume-se culpada até que a Defesa a redima.

Processo nº CC-COMPRAS-001 Réu: cockpit /compras Provas: compras-page.jsx · Index.tsx@main

Este auto não busca elogiar. A Promotoria recebeu ordem de tentar derrubar a tela — encontrar toda mentira de CTA, toda incoerência entre protótipo e produção, todo cromo decorativo e toda barreira de acessibilidade. A Defesa contra-argumenta com o melhor caso possível (steelman). O Juízo decide e fixa a pena — a ação concreta de conserto. Oito autos foram instaurados.

AUTO I

Botões que mentem

crime de falso testemunho de interface
Grave
Acusação

A tela exibe ações que não executa. ↓ Importar XML está disabled; CSV / Excel / PDF abrem o Blade legado /purchases em outra aba, quebrando a ilusão de app único; e o botão primário do drawer Executar → não diz o que executa. Botão que finge ação fere a regra de honestidade do projeto.

Defesa

São stubs honestos: o Importar XML tem title "Wave 6"; os exports são ponte temporária até endpoints nativos; "Executar →" é placeholder de mock, não produção.

Veredito Culpado em parte

Tooltip não absolve um botão que parece clicável e nada faz. O "Executar →" é indefensável.

Pena: "Executar →" passa a nomear o verbo real do estágio (Conferir itens · Marcar recebida · Pagar agora · Enviar pedido). Exports e Importar XML: estado disabled visível com tooltip, ou removidos até a Wave que os entrega — nunca abrindo aba para outro app sem aviso.

AUTO II

Dois corpos, uma só alma

crime de identidade dividida (drift protótipo × git)
Capital
Acusação

Existem duas telas Compras diferentes. O protótipo (compras-page.jsx) usa drawer overlay lateral estilo Venda e cabeçalho os-page-h canônico. O git (Index.tsx@main) usa drawer split inline e cabeçalho ad-hoc .hd. A charter promete "pin literal do protótipo" — promessa quebrada. Qual é a tela real?

Defesa

O loop F1→F3 prevê o protótipo (semente) à frente do git (implementação). Drift é a natureza do processo, não defeito.

Veredito Culpado

Drift tolerado vira dívida permanente. As duas versões já divergem em cabeçalho, drawer, KPI e ações.

Pena: ponte zero-toque pro [CL] reconciliar Index.tsx ao protótipo: drawer overlay + <PageHeader> canon + dropdown Ações + toolbar + ordenação + rodapé de totais. Uma alma só.

AUTO III

Cabeçalho herege

desacato à lei do cabeçalho único (ADR 0189/0190)
Média
Acusação

O git ainda renderiza .hd com breadcrumb "ERP · Operação · Compras" e <h1> de 16px — ignorando o <PageHeader> canônico (h1 22/700, borda quente, align center) que o resto do ERP já adotou em 2026-06-16.

Defesa

O .hd é anterior à convergência de cabeçalho; ainda renderiza, não quebra.

Veredito Culpado (no git)

"Renderiza" não é "conforme". Cinco cabeçalhos divergentes já foram erradicados; este é o sexto fugitivo.

Pena: migrar .hd → os-page-h / <PageHeaderPrimary>, crumbs pra fora, contagem vira subtítulo. (O protótipo já cumpriu.)

AUTO IV

O KPI de duas caras

indução do usuário a erro de leitura
Grave
Acusação

O card "A pagar" mostra R$ 14.221,30 no protótipo, mas 3 (contagem de compras) no git. Mesmo rótulo, duas semânticas. Larissa lê "A pagar: 3" e não sabe se são 3 compras ou 3 reais.

Defesa

São telas em waves diferentes; o git é scaffold mínimo, o protótipo é a meta.

Veredito Culpado

Ambiguidade num KPI de dinheiro é falha cara num balcão.

Pena: padronizar — valor em R$ como número-herói + subtítulo "N compras · próx. venc. dd/mm" nos dois lados. Número de dinheiro nunca disputa rótulo com contagem.

AUTO V

Filtros de vitrine

cromo decorativo simulando função
Média
Acusação

As pills Fornecedor: todos · Local: Matriz · Maio/2026 · Pagamento: todos não filtram nada — e uma delas ostenta o estado ativo .on como se estivesse aplicada. No git, a pill "Em breve" pelo menos confessa. O protótipo finge.

Defesa

Filtros server-side são Wave 7; as pills antecipam o layout final.

Veredito Culpado em parte

Antecipar layout é legítimo; fingir estado ativo num filtro morto não é.

Pena: ligar os filtros client-side (os dados já estão na lista) ou remover o .on e marcar visualmente como "em breve". Nada de pill acesa que não filtra.

AUTO VI

Ordenação inacessível

barreira de teclado (WCAG 2.1)
Grave
Acusação

Os cabeçalhos ordenáveis são <th onClick> — não focáveis por Tab, sem aria-sort, sem ativação por Enter. Larissa, que opera no teclado, não consegue ordenar. O leitor de tela não anuncia a direção.

Defesa

O mouse funciona; é um protótipo de avaliação visual.

Veredito Culpado

Acessibilidade não é polimento tardio — a persona-piloto é teclado-first.

Pena: <button> dentro do <th> + aria-sort="ascending|descending" na coluna ativa. Vale pro F3.5 [CA] também.

AUTO VII

O cockpit que não cria compra

abandono de contexto na criação (convergência C1)
Leve
Acusação

+ Nova compra teleporta o usuário pra /purchases/create — outra tela, outro módulo, outro visual. O cockpit chamado "Compras" não cria uma compra; terceiriza e some.

Defesa

É decisão arquitetural deliberada (ADR compras-purchase-convergencia-c1): /purchases é o canon de CRUD; duplicar o formulário seria a verdadeira falha.

Veredito Inocente, com ressalva

A arquitetura é sã. Mas a transição é silenciosa demais — parece bug de navegação.

Pena: sinalizar a costura — rótulo/tooltip "abre o cadastro completo (Purchase)" e transição visual coerente, pra a saída de contexto ser intencional, não susto.

AUTO VIII

A largura da Larissa

risco de densidade no monitor real (1280px)
Média
Acusação

No git, o drawer split reserva 480–540px; sobram ~700px pra lista de 9 colunas (Ação + 8). No alvo real da Larissa (1280px, balcão), isso espreme colunas ou força scroll horizontal — proibido pela meta de UX da charter.

Defesa

O protótipo já virou drawer overlay (não rouba largura da lista) e a visibilidade de colunas permite esconder o que não importa.

Veredito Culpado em suspeição

O overlay resolve no protótipo; o git ainda é réu. E "poder esconder coluna" ≠ "vir escondida".

Pena: testar a 1280px com drawer aberto; default ocultar colunas de baixa prioridade (NF-e/Itens) nesse breakpoint; portar o overlay pro git (ver Auto II).

Sentença

A tela sobrevive ao julgamento — mas sai com obrigações. O protótipo já cumpriu parte da pena; o git é o réu reincidente.
