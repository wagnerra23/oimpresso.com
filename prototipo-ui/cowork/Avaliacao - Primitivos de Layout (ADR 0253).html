Text
layout/text.tsx
size xs–3xl · weight · tone · align · truncate · as h1–h6/p/span
  • as desacopla semântica de estilo — h1–h6/p/span/label. Correto.

04 Falta alguma coisa? — sim, priorizado
P0
Text: mono + tabular + tons pos/neg + size até 4xl

É o gap mais caro: sem isso, nenhuma tela de número (vendas, financeiro, OS, NF-e) compõe com primitivos — cai de volta em classe solta, que é justamente o que a ADR quer matar. Adicionar 4 variantes ao CVA, custo baixo, destrava o maior volume de telas.

P0
Container: verificar max-w-screen-* no Tailwind v4

Se as utilities foram removidas (provável no TW v4), o limite de largura é silenciosamente nulo. Rodar build, inspecionar o DOM. Se confirmado, trocar por max-w-2xl/4xl/6xl… ou container query. Bug de uma linha, impacto em toda página.

P1
Grid: colunas responsivas + auto-fit/minmax

Prop tipo cols={{ base:1, md:2, xl:3 }} ou min="280px". Sem isso, dashboards e grades de card não migram de verdade — e migrar dashboard é metade do valor da camada.

P1
Box: prop de cor/superfície via token

A ADR nomeou "espaço/cor"; entregar bg (surface/sunken/accent-soft) + radius + border token fecha a promessa e cobre o "card" sem CSS. Senão o card vira o próximo CSS bespoke.

P2
Divider em Stack/Inline + primitivo de prova viva

divider (hairline/"·") cobre um padrão visual recorrente. E entregar a tela piloto 100% primitivos + doc no REGISTRY fecha o "critério de pronto" da própria ADR — hoje aberto/não verificado.

P2
Enforcement (follow-up assumido pela ADR)

Lint que proíbe flex/grid solto fora de Components/layout/. A ADR adia de propósito (precisa da camada + telas migradas antes), mas sem ele a camada não "pega" — vira opcional.
