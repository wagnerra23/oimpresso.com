03 — RESOLUÇÃO & HANDOFF

Um header, um pageheader — convergir para o canon do git

O canon do git é o lado com ADR (0189 v3.8 / 0190) e gates de CI. A recomendação é alinhar protótipo e as telas remanescentes do git ao componente único, dobrando para dentro do canon as evoluções recentes do protótipo que [W] quiser manter (peso, tempero).

O que o Code aplica (git @main)

Ordem proposta — cada item é um PR isolado, ratchet, para no gate visual.

H-1
Migrar telas legadas para <PageHeader>

Trocar headers inline os-page-h / fin-page-h / cli-pageheader que ainda existem (Financeiro/Unificado, DRE, Caixa Unificada) pelo componente canon. Retira a caixa de ícone e os tons crus.

H-2
Padronizar o primário no <PageHeaderPrimary>

Qualquer botão de criação na Zona R vira o roxo universal oklch(0.55 0.15 295), 32px, Plus stroke 1.75 — mata os .os-btn.primary por-tela.

H-3
Decisão [W] · header global

Eleger uma gramática de nav-de-módulo: breadcrumb-dropdown + topnav-chip (git atual) ou PageHeaderNav sublinhado (protótipo). Recomendação: manter o do git (já em produção, com hideTopbar), e o protótipo adota a mesma.

H-4
Decisão [W] · evolução do canon

Se [W] quiser as evoluções do protótipo no canon, vira amendment v3.9: peso H1 600 vs 700, tempero (--sh-1, --ease) no header. Tier 0 — só com OK de [W].

O que [CC] alinha no protótipo (minha pista)

Para o protótipo deixar de ser fonte de divergência.

P-1
Unificar os 3 headers do protótipo em um

Um só padrão (espelhando o canon): .cli-pageheader e o .os-page-h de cada tela passam a render igual — center, padding 24/24/14, peso conforme decisão H-4.

P-2
Erradicar oklch cru de .cli-pageheader

Tons danger/warn por token (var(--bad) / var(--warn)), não literal — para o protótipo passar pela mesma régua do gate.

Ponte zero-toque: o pacote H-1..H-4 será entregue como prototipo-ui-patch/PROMPT_PARA_CODE_HEADER-PAGEHEADER-CONVERGENCIA.md + URLs públicas (~1h) para o Claude Code aplicar via 1 colagem. Este documento é a especificação visual que acompanha o prompt. Nada está commitado — o Code resolve com o pedido; H-3/H-4 esperam decisão de [W].
