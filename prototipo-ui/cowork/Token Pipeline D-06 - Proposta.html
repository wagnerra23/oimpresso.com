D-06 Token Pipeline · Protótipo = Produção

Uma fonte de token. Dois consumidores. Zero retradução.

Hoje a mesma cor é declarada em 3 lugares que divergem — e o handoff [CC]→[CL] é uma retradução manual, exatamente onde a qualidade vaza. O D-06 propõe: a cor nasce uma vez, um build a distribui pro protótipo (Cowork) e pra produção (Laravel/Inertia), e os gates garantem que ninguém mais redeclara à mão.

Decisão: D-06 · _PROPOSTA-ds-harmonizacao.md Roxo canon oklch(0.55 0.15 295) · ADR 0235 — intocado PROPOSTA [CC] · Tier 0 · decide [W] · aplica [CL]

1 O problema hoje

A mesma cor mora em 3 arquivos — e eles escorregam

Cada um foi escrito à mão, numa época diferente. Quando o roxo muda num, os outros ficam pra trás. Foi o que gerou "verde×roxo", "azul 220 stale", o §16.2 que virou mentira.

PROTÓTIPO · Cowork
ds-v5/tokens.css
O que o host oimpresso.com.html carrega (linha 11). :root + dark + density.
PRODUÇÃO · Laravel/Inertia
resources/css/inertia.css @theme
O único @theme que gera as utilities Tailwind (bg-primary…). Congelado pelo foundation-guard.
MIRROR v4 · transporte
prototipo-ui/tokens.css
Cópia v4 que o Code baixa via curl. Outra fonte que precisa ser mantida em sincronia à mão.
⚠ Drift garantido: 3 fontes × manutenção manual = uma sempre fica velha. O handoff vira "retraduzir" cor de um arquivo pro outro — onde a qualidade vaza (D-06, _PROPOSTA-ds-harmonizacao.md).

Nasce 1 vez → build distribui → gates travam o resto

design-tokens/tokens.css
build

Um script transforma a fonte nos arquivos que cada lado já sabe consumir. Determinístico, versionável.
npm run tokens

Consumidor 1 · Protótipo

ds-v5/tokens.css
Consumidor 2 · Produção
Laravel/Inertia

O handoff [CC]→[CL] deixa de ser "retraduzir cor" e passa a ser "importar o gerado". A cor nunca mais é digitada duas vezes — logo nunca mais diverge.

4 Adoção em fases

Incremental — cada fase entrega sozinha, nada quebra atrás
FASE 1

Eleger a fonte

Promover ds-v5/tokens.css (já é o que o host usa, roxo canon) a design-tokens/tokens.css — a verdade. Nenhum valor muda; só ganha um endereço único.

sem risco · não toca produção
FASE 2

Build determinístico

Script npm run tokens que copia/transforma a fonte. Saída versionada no git → diff revisável, sem mágica.

aditivo · gate-safe
FASE 3

Produção importa

Criar foundations.css com o @theme gerado e dar @import no inertia.css. Baseline do inertia.css cai (53→~0). É Tier 0 — precisa de olho [W]/staging (gate de pixel ainda é STUB).

Tier 0 · só [W]
FASE 4

Protótipo importa

ds-v5/tokens.css passa a ser o @import do gerado. Cowork e produção lendo o MESMO arquivo. D-06 fechado.
