Receita de TelaDS v6

Como nascer uma tela.

A tela não se desenha do zero — ela se monta do kit. Esta receita é a ordem que faz qualquer tela nova já nascer coerente, bonita e em claro/escuro. Quem segue, passa no lint e na crítica sem retrabalho.

1

Comece pelo shell, não pela tela

Toda tela mora no Cockpit V2: sidebar (dark-fixo) · header · corpo · e rail de contexto ou drawer. Não reinvente o esqueleto — só preencha o corpo.

<!-- corpo da tela: container + tokens, nada de cor crua -->
<div class="minha-tela">
  <header> c-id (header de identidade) + ações </header>
  <div class="kpis"> c-kpi × N </div>
  <main> conteúdo + c-tl / c-asset ... </main>
  <aside> c-rail + c-nba (contexto / Jana) </aside>
</div>
2

Monte com o kit — não escreva CSS de cor

Pegue as peças do Showcase. Se precisa de algo que o kit não tem, é sinal de buraco no DS (passo 5), não de CSS novo.

  • ✓
    Header de identidade .c-id — cliente/frota/OS no topo.
  • ✓
    KPI / stat .c-kpi — número grande, tom semântico quando importa.
  • ✓
    Pílula de status .c-pill + etapas .c-stage.
  • ✓
    Timeline unificada .c-tl · card de ativo .c-asset.
  • ✓
    Rail de contexto .c-rail + próxima-ação .c-nba.
3

Obedeça os tokens — o claro/escuro sai de graça

Cor = tokenSó --accent, --pos/neg/warn, --origin-*, --stage-*. O roxo 295 é a marca.
Superfície em camada--bg → sunken → surface → raised. Borda sempre --border.
Nunca oklch cruCor escrita à mão não flipa no tema e quebra o lint. Zero exceções.
Nunca modal pra detalheDetalhe = rail ou drawer lateral. Densidade pro 1280 da Larissa.
4

Pense em fluxo, não em tela

Pergunte: de onde o dado vem e pra onde ele vai? Desenhe a costura (a passagem pro próximo módulo) — é onde o ERP ganha ou perde. Surfe o contexto cross-módulo no rail.

5

Faltou peça? É buraco do DS — proponha, não invente

Esse é o ciclo que mantém o sistema vivo (foi assim que a paleta --stage-* nasceu):

monto a tela→ sobra cor/peça crua→ isso é buraco do DS→ vira token/componente→ harmonizo

Mudança no DS é Tier-0 (decisão de [W], vira ADR). Eu proponho; nunca cunho versão sozinho.
