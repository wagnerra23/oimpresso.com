F1 — Aba “Preço especial” · modelo de grade + regra + exceção

O modelo de grade define os eixos (`variation_templates`, que já existe no schema com CRUD vivo desde 2017). A lista de preço é uma regra (%), não uma grade de digitação: a matriz mostra o preço efetivo calculado, e digitar numa célula cria uma exceção. Padrão convergente de Shopify B2B · Tiny/Olist · Bling · Odoo.

  • Troque o modelo (Tamanho / Cor / Numeração / Voltagem) — a grade inteira se refaz.
  • Desmarque um valor (chip) — ele some da grade e o preview recalcula.
  • Clique numa célula hachurada pra reativar a combinação que “não existe”.
  • Clique nas listas (Varejo / Atacado / Revenda) — o preço recalcula pela regra de cada uma.
  • Digite numa célula → vira exceção (negrito + tarja + contador na aba). O ↺ desfaz.
  • Mude o % da regra → o herdado recalcula; a exceção não se mexe (o invariante).
  • Teclado: Tab anda coluna · Enter pula linha (padrão Cin7/Lightspeed).
