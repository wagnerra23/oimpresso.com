F1 — Aba “Preço especial” · modelo de grade + regra + exceção

Preço base (fonte, digitado — mora na variação) + tabelas (regra %, calculada). O produto pode ter só variação, só tabela ou os dois — é a mesma tela: muda quantas linhas a grade tem e quantas abas de tabela existem. Produto simples = grade de 1 célula (a variação DUMMY). Padrão convergente de Shopify B2B · Tiny/Olist · Bling · Odoo.

  • Aba “Preço base” → o preço próprio de cada variação. Azul-G nasce 120 (os outros 100) de propósito: a base varia por célula.
  • “Atacado” = regra % (−20%) → Azul-P vira 80,00 e Azul-G vira 96,00. Uma regra só, bases diferentes.
  • “Revenda” = preço definido por produto — sem %: o cliente digita o valor na tabela (o modo “Customizada” do Bling). Célula não digitada usa o preço base.
  • Troque o modo no seletor da tabela — o mesmo valor deixa de ser “exceção com tarja” e vira “o preço” (neutro), porque é isso que ele é.
  • Edite a base e volte à tabela → o preço da lista segue sozinho.
  • Digite numa célula de tabela → vira exceção (negrito + tarja + o calc. embaixo mostra de onde desviou). O ↺ desfaz.
  • Mude o % → o herdado recalcula; a exceção não se mexe (o invariante).
  • Troque o modelo de grade (Tamanho / Cor / Numeração / Voltagem) — a grade se refaz. Desmarque um chip → some da grade.
  • Clique numa célula hachurada pra reativar a combinação que “não existe”.
  • Teclado: Tab anda coluna · Enter pula linha (padrão Cin7/Lightspeed).
