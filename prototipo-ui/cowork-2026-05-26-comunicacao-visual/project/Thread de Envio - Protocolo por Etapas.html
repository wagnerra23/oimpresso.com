Como rodar a thread

Ordem de colagem: Msg 1 + Msg 2 + Msg 3 podem ir juntas (domínios disjuntos). Espere o Portão B (F1) mergear → cole Msg 4. Clientes passou limpo → cole Msg 5. Portão A congelado → cole Msg 6. Ports fecharam → cole Msg 7.

Você nunca interpreta nada — cada mensagem é um bloco fechado. O Code abre os PRs; você revisa e mergeia no seu ritmo (o F2 é o gargalo, não o código).

Limite honesto

Eu não escrevo no git. Esta thread é o conteúdo pronto; o transporte é seu (cola no Code). Duas mensagens têm parte que só você faz: Msg 1 (congelar — sua assinatura) e os merges de cada PR (F2). Sem o congelamento da Msg 1, as Msgs 6 ficam presas.

Base: plano de execução paralela + ficha por onda (detalhe+adversário) + reconciliação [CL] @main. As "regras duras" da Msg 5 vieram das farpas do [CX] (E2E Vendas, A0 Financeiro, ui/ read-only) — o adversário já está dentro das mensagens, não fora.

Honestidade: formato espelha o protocolo zero-toque (COWORK_NOTES→F0, 1 PR/tarefa, soberania [W]). Não é execução — é o pacote de envio. Nada roda até você colar + congelar.
