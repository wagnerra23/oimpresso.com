# Onda 1.3 — Régua por tela (Sells)

> Status vivo do programa: [PLANO-MESTRE.md](../PLANO-MESTRE.md) §Status vivo (1 plano = 1 registro) — execução via tasks MCP `parent_plan=programa-ondas`.

## Objetivo

Aplicar a régua estendida (0b) às telas de venda — a primeira foto que mostra UX **e**
comportamento juntos.

## Telas no escopo

- `resources/js/Pages/Sells/Create.tsx` (persona Larissa — a mais crítica)
- `resources/js/Pages/Sells/Edit.tsx`
- `resources/js/Pages/Sells/Show.tsx`
- `resources/js/Pages/Sells/Index.tsx`

## Passos

Para cada tela:
1. `screen-grade` (UX, 16 dimensões, LLM-as-judge, persona-weighted) — **inalterado**.
2. **`casos_coverage`** — cruzar `.casos.md` da tela ↔ testes reais (via `casos-coverage-guard.mjs`); listar os UC-IDs que a defendem + status.
3. **Dente de cálculo (D1)** onde toca valor (`Create`/`Edit`/`Show` tocam `final_total`, desconto, pagamento).
4. Gravar o scorecard estendido em `memory/governance/scorecards/screens/sells-<tela>.yaml`.

## ⚠️ Armadilha do casos-gate (ordem 1.3 ↔ 1.4)

O `casos-gate` é **required** e a regra G-2 diz "UC declarado sem teste = órfão". Se esta
etapa **declarar** UCs novos nos `.casos.md` de Sells antes da 1.4 escrever os testes, o
próprio programa quebra o CI — foi exatamente por isso que a `/perfil` parkou comportamento
em "backlog sem id". **Ordem segura:** nesta etapa, os UCs sem teste entram no scorecard
(`casos_coverage` status ⬜) e permanecem no backlog-sem-id do `.casos.md`; eles só ganham
id de UC **no mesmo PR** que traz o teste (1.4). Declarar UC + teste = 1 PR, sempre.

## Critério de pronto

- 4 scorecards com **UX + cobertura de comportamento** lado a lado.
- Gaps rankeados por exposição Tier-0.

## Verificação

- `sells-create.yaml` mostra nota **alta de UX** (~90) **e** `casos_coverage` **baixo** — a
  contradição que hoje é invisível e que justifica o programa inteiro.

## Resultado esperado (a prova visual)

O scorecard de `Sells/Create` vai gritar: tela premium de UX, cálculo de venda sem rede. É a
`/perfil` de novo, mas na tela que roda 50×/dia no cliente que é 99% do volume.
