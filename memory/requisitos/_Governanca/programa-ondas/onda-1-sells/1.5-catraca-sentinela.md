# Onda 1.5 — Catraca + sentinela (Sells)

> Status vivo do programa: [PLANO-MESTRE.md](../PLANO-MESTRE.md) §Status vivo (1 plano = 1 registro) — execução via tasks MCP `parent_plan=programa-ondas`.

## Objetivo

Travar o que a Onda 1 ganhou pra não regredir — o pilar **catraca** aplicado a Sells, e a
**sentinela** passando a vigiar o módulo.

## Passos

1. **Registrar novos baselines** pós-onda:
   - `screen-grade` das telas Sells (scorecards estendidos da 1.3).
   - `casos_coverage` (os UCs que passaram a ser defendidos por teste).
   - `exposicao-tier0` (o débito de Sells depois do dente de cálculo).
2. **`casos-gate` passa a defender** os UCs de Sells que ganharam teste (1.4) — regressão bloqueia PR.
3. **Sentinela vigia Sells** — o cron semanal (0c) inclui Sells no conjunto quente.

## Critério de pronto

- A catraca **falha** se qualquer camada de Sells regredir (nota UX, cobertura de comportamento, ou débito de exposição).

## Verificação

- Simular regressão (remover um teste de cálculo, ou baixar a cobertura de um UC) → CI bloqueia o PR.
- O baseline pós-onda vira o novo piso de Sells.

### Critério de pronto — PROVADO (evidência executada)
- **"Simular regressão (remover um teste de cálculo) → CI bloqueia"** — removi `tests/Feature/Calculo/CalculoValorSellsTest.php` e rodei o guard:
  ```
  $ node scripts/casos-coverage-guard.mjs   → exit 1
    🆕 uc-orphan:resources/js/Pages/Sells/Create.casos.md#UC-S02
    UC NOVO precisa de teste citando o id (ADR 0264 G-2).
  $ (restaurado o arquivo) node scripts/casos-coverage-guard.mjs → exit 0
  ```
- **"A catraca falha se qualquer camada de Sells regredir"** — `node scripts/qa/exposicao-tier0.mjs --check` → `exit 0` ("nenhuma regressão do conjunto quente"); `Sells/Create.tsx`+`Sells/Index.tsx` no `covered_hot_screens`. Camadas: comportamento/UC → `casos-gate` (G-2, UC-granular); exposição/cobertura de tela → `exposicao-tier0 --check` (screen-granular); UX → ratchet de screen-grade.

### Nuance honesta (camadas, não gap)
`exposicao-tier0 --check` protege por **tela** (`covered_hot_screens`); `Sells/Create` também conta
como coberta via **UC-S01** (e2e). Logo, remover **só** o teste de cálculo NÃO dispara a sentinela
de exposição (a tela segue coberta por UC-S01) — mas **dispara o casos-gate** (UC-S02 órfão, prova
acima). O dente de cálculo está defendido pela camada certa (rastreabilidade UC), não pela de
exposição. Ortogonais, cada uma no seu papel — sem buraco.
