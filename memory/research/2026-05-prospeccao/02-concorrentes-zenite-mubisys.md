# Concorrentes — Zênite Sistemas & Mubisys (Mubi Sistemas)

> Pesquisa competitiva 2026-05-09. Fontes: site oficial dos concorrentes, Reclame Aqui, LinkedIn, CNPJ.biz, Econodata, imprensa setorial (Singrafs/Assingrafs/Infosign/Guia do Gráfico). Apenas conteúdo público — sem engenharia social ou criação de conta em demos.
>
> **Diferenciais oimpresso usados como contraste:** NFe automática a partir de boleto pago (US-RB-044); Jana IA com memória persistente (Meilisearch hybrid); Multi-tenant Tier 0 isolation; Stack Laravel 13.6 + Inertia v3 + React 19 + Tailwind 4; Governança formal Constituição v2 (ADR 0094).

# Zênite Sistemas

## Identidade
- **Site:** http://zenitesistemas.net.br/ + https://www.zsl.com.br/ (site institucional/comercial moderno)
- **HQ:** Belo Horizonte/MG — Rua Itaguai, 866, Caiçaras, 30.775-110 ([Econodata/CNPJ.biz](https://cnpj.biz/00219676000170))
- **CNPJ:** 00.219.676/0001-70 (fundada **1994** — 32 anos de mercado)
- **Tempo de mercado:** "30+ anos" (alegação no site oficial — coerente com fundação 1994)
- **Modelo:** Hybrid — produto principal **GE 4.0 (GWorks Enterprise)** com versão **web** (recente) + base instalada legacy desktop. Reclamações Reclame Aqui mencionam "alteração para versão web após quase 3 anos usando o sistema" → migração desktop→web em curso, ainda não 100% SaaS-cloud nativo.
- **Pricing público:** **Não publicado.** Vendas via comercial (vendas@zsl.com.br / +55 (31) 3419-7300). Tiers nomeados: GE Lite, GE Smart, GE Standard, GE Pro/Full.
- **Tamanho estimado:** **11-50 funcionários** ([LinkedIn](https://br.linkedin.com/company/z-nite-sistemas)). Alega **2.200+ gráficas atendidas** Brasil afora. Parte do "Grupo DWG".

## Stack/UX (do que dá pra ver publicamente)
- **Tecnologia aparente:** sistema legado em modernização. Versão web ainda apresenta instabilidades em finais de semana (reclamação histórica Reclame Aqui — empresa rebate alegando SLA 99%). Não há indicação de stack moderna (React/SPA) — o login intranet (`https://intranet.zsl.com.br`) sugere portal corporativo tradicional.
- **UI:** Não publicam screenshots no marketing site novo (zsl.com.br). Site institucional usa copy verbo-de-ação ("Controle total para quem cria, imprime, fatura e entrega"). Material de imprensa (Singrafs 2016/2017) mostra UI desktop tradicional.
- **Mobile:** **Não há app mobile mencionado no site nem na Apple App Store / Google Play.**
- **Modules/features destacados:**
  - Mapa RKW (custos reais)
  - Orçamentos
  - OS/PCP (Produção)
  - Financeiro (AR/AP)
  - Estoque
  - NFe + SPED
  - CRM
  - Coleta automática de dados de máquinas (production data collection)
  - Integração WhatsApp (orçamento/nota/cobrança)
  - Multi-empresa
  - BI/Relatórios customizáveis
  - **Módulo Balcão** (lançado pra atender frente de loja com vendas expressas + papelaria — [Guia do Gráfico](https://www.guiadografico.com.br/noticias/zenite-sistemas-lanca-novo-modulo-balcao-para-o-ge-30))

## Features alegadas
- ERP completo end-to-end pra gráfica
- **Coleta automática de dados das máquinas** (diferencial relevante — IoT/integração equipamentos)
- Multi-empresa nativo
- 4 tiers segmentados por porte
- Segmentos: promocional, editorial, embalagens, filmes flexíveis, rótulos, jornal, sinalização, large format, serigrafia, digital, acabamento, pré-impressão, balcão, brindes (cobertura ampla — concorre direto com oimpresso em sinalização/large format/brindes)
- Migração desktop→web ativa (modernização em andamento)

- **Reclamação concreta encontrada:** [`zenite sistemas, uma pessima escolha para comunicação visual!`](https://www.reclameaqui.com.br/zenite-sistemas/zenite-sistemas-uma-pessima-escolha-para-comunicacao-visual_EmZx7mAtts45a2LE/) — cliente desde 2014, queixa que **versão web nunca funciona aos finais de semana**. Empresa responde citando SLA 99%.
- **G2/Capterra/Trustpilot:** **sem páginas dedicadas encontradas** — não estão inventoried internacionalmente.
- **LinkedIn:** página existe mas com baixa atividade. Imprensa setorial (Singrafs, Assingrafs, Infosign, Revista Desktop, Guia do Gráfico) cita a empresa como referência tradicional do setor.
- **Depoimentos no site:** 4 depoimentos longos (clientes 25+ anos) — tom "parceria de longa data", "evolução do sistema", "acessível" — sinal de **base fiel mas envelhecida**.

- **Como vendem:** demo comercial (não há trial autoatendimento). Site direciona pra contato.

- **Setup fee/treinamento:** não publicado. Padrão do setor brasileiro nesse segmento é cobrar implantação separada.

## Gaps explorávveis vs oimpresso
**O que oimpresso tem e Zênite NÃO mostra ter:**
1. **NFe automática a partir de boleto pago (US-RB-044)** — Zênite tem NFe + SPED, mas não publica fluxo "boleto pago → NFe emitida sem clique humano". Oimpresso tem.
2. **IA conversacional com memória persistente (Jana)** — Zênite tem WhatsApp e BI, mas zero menção de IA/agente conversacional. Oimpresso tem stack canônica (laravel/ai 0.6.3 + Meilisearch hybrid + 4 Agents próprios).
3. **Stack moderna front-end** — Zênite ainda migra desktop→web; oimpresso já é Inertia v3 + React 19 + Tailwind 4 nativo cloud.
4. **App mobile nativo** — Zênite não tem; oimpresso tem caminho aberto pra PWA via Inertia.
5. **Governança formal pública** — Zênite não publica ADRs, processo de mudança, SLA detalhado; oimpresso tem Constituição v2.

**O que Zênite tem e oimpresso NÃO tem (perigo):**
1. **Coleta automática de dados das máquinas (IoT/integração equipamentos)** — produção real-time vinda do equipamento. Oimpresso não tem isso destacado em SPEC. **Risco competitivo médio-alto** em prospects com parque de máquinas grandes.
2. **Mapa RKW (custos por método alemão)** — metodologia de custeio sofisticada que clientes industriais conhecem. Oimpresso tem Financeiro mas não publiciza RKW.
3. **Base instalada de 2.200 gráficas** — efeito rede / referenciabilidade. Oimpresso tem 7 clientes ativos (ROTA LIVRE 99% volume — vestuário SC, Modules/Vestuario; outros 6 são pilotos OfficeImpresso legacy a migrar pra Modules/ComunicacaoVisual).
4. **Multi-empresa nativo** publicizado — oimpresso tem multi-tenant Tier 0 mas o framing comercial é diferente (multi-tenant ≠ multi-empresa do mesmo dono no UI).
5. **30 anos de mercado + endorsement Singrafs/Assingrafs** (associações setoriais) — credibilidade institucional.

**Onde oimpresso pode atacar especificamente:**
- "Zênite faz NFe, mas você ainda clica pra emitir depois que o boleto cai. No oimpresso, o boleto pago dispara a NFe automática" — diferencial concreto demonstrável.
- "Zênite ainda está migrando o sistema antigo pra web — o oimpresso nasceu cloud, com a stack que o Mercado Livre e Nubank usam (React 19/Inertia)" — ataque na modernização inacabada.
- **Calcanhar de Aquiles:** instabilidade fim-de-semana documentada publicamente no Reclame Aqui — gráfica que opera Sex/Sáb (frente de loja Balcão) sente isso.

## Posicionamento sugerido pra prospect que usa Zênite há 5+ anos
**Pitch 30s:**
> "Você já tem o ERP que cobre orçamento → produção → NFe. O que mudou nos últimos 2 anos é que dá pra fechar o loop financeiro: o boleto pago no Asaas/Inter dispara a NFe automática, sem alguém clicando. E a Jana — IA do oimpresso — sabe responder 'qual cliente está atrasado', 'qual produto mais lucrou em abril', sem você abrir relatório. Não é trocar o que funciona — é eliminar os 2 cliques que sobraram."

**3 perguntas SPIN pra abrir dor:**
1. **Situação:** "Quantas notas vocês emitem por dia hoje? Quem é a pessoa que clica em 'emitir NFe' depois que o cliente paga?"
2. **Problema:** "Quando o sistema fica fora do ar no fim de semana — sei que Zênite teve esse histórico — o que acontece com a frente de loja?"
3. **Implicação:** "Se a Larissa do balcão pudesse perguntar 'quanto a gente faturou semana passada' direto no chat e receber resposta em 3 segundos, em vez de pedir relatório pro financeiro, quanto isso vale por mês?"

# Mubisys (Mubi Sistemas)

## Identidade
- **Site:** https://mubisys.com/ (institucional) + https://lp.mubisys.com/ + https://lp2.mubisys.com/ (landing pages)
- **HQ:** **Barueri/SP** — Avenida Trindade 254, salas 215/216, 06404-326 ([CNPJ.biz](https://cnpj.biz/17673962000104))
- **Razão social:** Mubi - Tecnologia da Informação LTDA
- **CNPJ:** 17.673.962/0001-04
- **Tempo de mercado:** **fundada 15/02/2013** — ~13 anos
- **Modelo:** **SaaS cloud puro** ("totalmente online", "controle de onde estiver"). Tem **app mobile iOS + Android sem custo adicional** (App Store + Google Play).
- **Pricing público:** **Não publicado.** Vendas via apresentação comercial → proposta. **Trial gratuito de 7 dias** (mencionado em termos de uso).
- **Tamanho estimado:** indeterminado pelos públicos do LinkedIn ([linkedin.com/company/mubi-sistema-de-gestao](https://br.linkedin.com/company/mubi-sistema-de-gestao)). Site alega **14.000+ usuários ativos / 1.800+ empresas ativas / 150+ TB de arquivos** — métricas "tração" fortes. RH em rh@mubisys.com.

## Stack/UX
- **Tecnologia aparente:** SaaS web + apps mobile nativos (iOS/Android). Não publica stack tech, mas o site `mubisys.com/index.php` ainda usa querystring com `index.php?app=` → backend PHP tradicional, possivelmente Laravel ou framework similar. Não está em stack ultra-moderna como React 19/Inertia.
- **UI:** copy "telas simples e objetivas", "interface intuitiva". Não publica screenshots claras no site principal — vídeo no YouTube ("Conheça o Mubisys em 2 minutos") mostra UI funcional mas tradicional.
- **Mobile:** **app dedicado iOS+Android** (br.com.mubi no Google Play / id1623941063 na App Store). Reviews indicam que **algumas operações são PC-only** — mobile é complementar, não paridade.
- **Modules/features destacados:**
  - **Comercial:** funil de vendas, orçamentos, envio via e-mail/WhatsApp, "termômetro de acesso" (engagement tracking), conversão pra OS, comissões
  - **Financeiro:** receitas/despesas, boletos, conciliação bancária, cash-flow forecasting
  - **Estoque:** alertas de mínimo, sistema integrado de compras com cotações
  - **NFe + NFCe + MDFe + NFSe + Sintegra + SPED** (cobertura fiscal completa)
  - **PCP/Produção:** gráficos de conclusão, sectorização, relatórios
  - **MubiDrive:** DAM (digital asset management) — clientes enviam arquivos direto pro sistema. 5GB grátis + ilimitado pra arquivos de visualização. **Diferencial forte.**
  - App mobile iOS/Android

## Features alegadas
- ERP 100% focado em comunicação visual (não é genérico adaptado)
- 14k+ usuários, 1.8k+ empresas, 150+ TB de arquivos, R$ [redacted Tier 0]M+ em economia gerada
- 4.9/5 estrelas em 300+ reviews (próprio site)
- Líder em software de gestão pra Comunicação Visual (Instagram oficial)
- Trial 7 dias gratuito
- Suporte direto na plataforma (chat) — sem telefone enfatizado (queixa de cliente)

## Reviews/percepção pública
- **Reclame Aqui:** ["sem reputação definida"](https://www.reclameaqui.com.br/empresa/mubi-sistemas/) — não atinge o mínimo de 10 reclamações avaliadas. Período 09/2025–02/2026 sem reclamações novas. **Empresa há 6+ anos no Reclame Aqui.**
- **Reclamações concretas encontradas:**
  1. ["Péssimo atendimento"](https://www.reclameaqui.com.br/mubi-sistemas/pessimo-atendimento_vB1PAXzlX1WND51q/) — atendimento por telefone difícil, empresa empurra chat e vídeos.
  2. ["Sistema engessado sem possibilidade de integração ou consulta de dados. Bom apenas para empresas gráficas de pequeno porte"](https://www.reclameaqui.com.br/mubi-sistemas/sistema-engessado-sem-possibilidade-de-integracao-ou-consulta-de-dados-bom_PlMWx0_YDQRfinlQ/) (fev/2023) — **gap crítico documentado: falta de API/integração e teto de porte da empresa.**
- **G2/Capterra:** sem páginas dedicadas.
- **App Store/Play Store:** apps existem (Mubisys id1623941063 / br.com.mubi). Não consegui extrair nota agregada via search.
- **Instagram:** ativo (@mubisys), tom institucional/comercial.
- **YouTube:** canal ativo (@mubisys) com tutoriais — sinal de investimento em onboarding self-service.
- **AFACOM (associação Goiás):** Mubisys tem programa de implantação via AFACOM+ ([afacomplus.com.br](https://afacomplus.com.br/afacomplus)) — endorsement setorial regional.

## Pricing & comercial
- **Como vendem:** apresentação comercial → proposta customizada. **Trial 7 dias gratuito** disponível.
- **Sinais de tier:** não publicados. Termos de uso descrevem "PROPOSTA COMERCIAL contendo condições e valores" — modelo enterprise com customização.
- **Setup fee/treinamento:** não publicado. AFACOM+ promove implantação assistida regional.

## Gaps explorávveis vs oimpresso
**O que oimpresso tem e Mubisys NÃO mostra ter:**
1. **NFe automática a partir de boleto pago** — Mubisys emite NFe + boleto + concilia, mas não publica trigger automático "boleto pago → NFe gerada".
2. **IA conversacional com memória persistente (Jana)** — Mubisys tem "termômetro de acesso" (engagement) mas zero IA/agente. Oimpresso tem stack canônica.
3. **API aberta / integrações de terceiros** — **reclamação Reclame Aqui documenta literalmente "sem possibilidade de integração ou consulta de dados"** (fev/2023). Oimpresso pode publicizar API + Webhook.
4. **Stack moderna React 19/Inertia v3** — Mubisys ainda é PHP tradicional (`?app=` na URL). Oimpresso é nativo SPA cloud.
5. **Governança formal Constituição v2** — Mubisys não publica processo de decisão, ADRs, SLA detalhado.

**O que Mubisys tem e oimpresso NÃO tem (perigo):**
1. **MubiDrive (DAM integrado)** — clientes enviam arquivos diretos no sistema, 150+ TB armazenados. Oimpresso **não tem DAM nativo** — gráficas trabalham com arquivos pesados (PDFs print-ready, plotagens) e isso é real-pain.
2. **App mobile iOS+Android maduro** — Mubisys tem ambos publicados. Oimpresso ainda é cloud-web responsivo.
3. **14.000+ usuários ativos** (alegado) — efeito rede e referenciabilidade.
4. **Cobertura fiscal mais ampla** publicamente — NFe + NFCe + MDFe + NFSe + Sintegra + SPED listados explicitamente. Oimpresso tem NFCe estado-1 (biz=1 Wagner SC) mas precisa publicizar checklist fiscal.
5. **Programa de canal/parceiro regional** (AFACOM+) — ecossistema de implantação distribuído. Oimpresso opera direto.
6. **Sales funnel + comissões nativos no Comercial** — Mubisys publiciza explicitamente; oimpresso tem mas não é destaque comercial.

**Onde oimpresso pode atacar especificamente:**
- "Mubisys tem reclamação pública (fev/2023) de cliente médio porte dizendo que **não dá pra integrar** com nada externo. O oimpresso nasceu com API + webhook." — ataque direto no calcanhar documentado.
- "Mubisys atende até pequeno porte bem; quando você cresce e precisa de API/dados externos, o sistema engessa. Oimpresso é multi-tenant Tier 0 com API aberta desde o nascimento."
- "Quando você quer perguntar 'o que mais lucrou em abril' no Mubisys, você abre relatório. No oimpresso, você pergunta pra Jana no chat e recebe a resposta com a query SQL auditável."

## Posicionamento sugerido pra prospect que usa Mubisys
**Pitch 30s:**
> "Mubisys cobre o básico bem — orçamento, financeiro, NFe, app mobile. Onde o oimpresso entra: quando você precisa cruzar dado, integrar com banco/Asaas/CRM externo, ou conversar com a IA sobre o seu próprio negócio. Tem reclamação pública dizendo que Mubisys engessa quando a empresa cresce — o oimpresso foi feito multi-tenant API-first desde o dia 1, e tem IA com memória que sabe falar do seu faturamento."

**3 perguntas SPIN pra abrir dor:**
1. **Situação:** "Hoje, quando você precisa cruzar dado do Mubisys com o seu banco ou com uma planilha externa, como faz? Exporta CSV?"
2. **Problema:** "Quando a empresa cresceu de X pra 2X, o Mubisys segurou? Já apareceu 'isso não dá pra fazer aqui'?"
3. **Implicação:** "Se a sua equipe pudesse perguntar pra IA — 'me mostra os 5 clientes que mais atrasaram boleto nos últimos 90 dias' — em vez de pedir pro TI montar relatório, quantas horas/semana isso libera?"
