# Sessão 11 — 2026-04-23

## Contexto

Wagner pediu para **conferir arquitetura** e, em seguida, fez uma checagem preventiva antes de atualizar o Laravel (10 → 11), com intenção final de chegar ao **Laravel 13** mantendo o sistema funcionando e priorizando **IA-first**.

## O que foi verificado (fatos do repo local)

- `composer.json` na raiz indica **Laravel 10**: `"laravel/framework": "^10.0"`.
- PHP local (Herd) está em **8.4.x**.

## Resultado do diagnóstico de upgrade (Composer)

### Bloqueadores atuais para Laravel 11

Saída do `composer why-not laravel/framework 11.*` mostrou bloqueios por versões atuais:

- `openai-php/laravel` (v0.4.3) **não aceita** Laravel 11 (só ^9.46|^10.7.1).
- `nunomaduro/collision` (v7.12.0) **conflita** com Laravel >= 11.
- `yajra/laravel-datatables-oracle` (v10.11.4) amarra `illuminate/*` em ^9|^10.
- `spatie/laravel-signal-aware-command` (1.3.0) amarra `illuminate/contracts` até ^10.

### Meta “Laravel 13”

`composer why-not laravel/framework 13.*` indicou que **Laravel 13 é bloqueado hoje** por limites superiores em tooling/libs (ex.: Inertia e Pest plugin), além da necessidade de upgrades grandes (Symfony 7/8, Carbon 3, Termwind 2, etc.).

Em particular:
- `inertiajs/inertia-laravel v1.3.4` suporta **até Laravel 12**.
- `pestphp/pest-plugin-laravel v2.4.0` suporta **até Laravel 11**.

### Checagens “é só major antiga?” (mesmo caso do Yajra)

Foi validado que vários pacotes são “mesmo caso”: **a major nova existe e acompanha Laravel mais novo**, mas a versão atual está defasada:

- **Yajra datatables**:
  - v10.11.4 → `illuminate/* ^9|^10`
  - v12.x → `illuminate/* ^12`
  - v13.0 → `illuminate/* ^13`
  - (Dry-run de v12/v13 falha no L10 porque requer subir o framework junto, o que é esperado.)

- **OpenAI Laravel (IA-first)**:
  - Dry-run mostrou que `openai-php/laravel v0.19.0` exige `laravel/framework ^11.29|^12.12`.
  - Conclusão: para IA-first, o caminho natural é **subir para Laravel 11/12** e então atualizar `openai-php/laravel`.

- **Pest plugin do Laravel**:
  - Dry-run mostrou que `pestphp/pest-plugin-laravel v4.0.0` exige `laravel/framework ^11.45.2|^12.25.0`.

- **DomPDF wrapper**:
  - Dry-run confirmou que `barryvdh/laravel-dompdf:^3` **atualiza com sucesso ainda no Laravel 10**.

### Observação sobre constraints

- Para `nunomaduro/collision`, o teste com constraint “seco” (`8`) falhou por ficar “apertado” (8.0.0). O correto é usar `^8` (ou `^8.5`) e manter em `require-dev` via `--dev`.

## Próximos passos sugeridos (IA-first + manter sistema funcionando)

1. Subir dependências “travadoras” ainda no Laravel 10 onde possível (ex.: `barryvdh/laravel-dompdf:^3`).
2. Planejar o salto **10 → 11** como etapa intermediária obrigatória para destravar `openai-php/laravel` moderno.
3. Depois, **11 → 12** (onde Inertia ainda suporta).
4. **13** só quando o ecossistema do projeto (principalmente Inertia + tooling) suportar — hoje o teto é 12 pelo adapter do Inertia.
