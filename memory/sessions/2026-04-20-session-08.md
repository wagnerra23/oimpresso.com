## O que foi feito

### 1. MarcacaoService (novo) — hash chain SHA-256

`Modules/PontoWr2/Services/MarcacaoService.php` — centraliza criação de marcações. Responsabilidades:

- `registrar(array $dados)` — dentro de `DB::transaction`:
  - NSR sequencial via `NsrService` (lock pessimista por REP_id)
  - Busca `hash_anterior` da última marcação daquele REP
  - `payloadCanonico($dados)` → `sha256($payload)` = `hash_atual`
  - `Marcacao::create(array_merge($dados, [...]))`
- `anular(Marcacao $original, $usuarioId, $motivo)` — não modifica a original (append-only). Cria uma **nova** marcação com `tipo=ANULACAO`, `marcacao_anulada_id=$original->id`, novo NSR + novo hash. Retorna a marcação de anulação.
- `verificarIntegridade($repId)` — varre marcações por NSR crescente, recomputa cada hash, retorna primeira divergência (ou null).
- `payloadCanonico($d)` — formato `business_id|colaborador_config_id|rep_id|nsr|momento|origem|tipo|hash_anterior|usuario_criador_id`; normaliza `momento` para `'Y-m-d H:i:s'`.

**Padrão PHP 7.1 usado:** `$self = $this;` + closure — `DB::transaction(function () use ($self, $dados) { ... })`. Closures em PHP 7.1 não têm `$this` auto-bound sem `Closure::bind`, e usar `$this` direto em closure passada para `transaction` gera "Using $this when not in object context".

**Consequências:**

- `Marcacao::anular()` agora delega: `return app(MarcacaoService::class)->anular($this, $usuarioId, $motivo);`
- Registrado como singleton no `PontoWr2ServiceProvider::register()`.

### 2. AfdParserService (rewrite) — Portaria 671/2021 funcional

`Modules/PontoWr2/Services/AfdParserService.php` — parser de ponta a ponta do arquivo AFD.

Tipos tratados:

- **Tipo 1 (header)**: CNPJ/CPF extraído por slice posicional; identificador REP (17 chars, pos 219); `Rep::firstOrCreate` por `(business_id, identificador)`.
- **Tipo 3 (marcação)**: PIS → `Colaborador::where('pis', $pis)->where('business_id', $bid)->first()`; dedup por `(rep_id, nsr)` — reimport idempotente; tipo inferido pela sequência do dia (`0=ENTRADA`, `1=ALMOCO_INICIO`, `2=ALMOCO_FIM`, `3=SAIDA`); delega registro para `MarcacaoService::registrar`.
- **Tipo 4 (retificação anterior)**: reutiliza `parseMarcacao`.
- **Tipo 5 (retificação posterior)**: idem.

Progresso incremental: `Importacao->update(['linhas_processadas' => N])` a cada 100 linhas.

Try/catch por linha — coleta até 20 amostras de erro em `erro_mensagem`; marca `ESTADO_CONCLUIDO` ou `ESTADO_FALHOU` conforme taxa.

Constructor: `public function __construct(MarcacaoService $marcacao) { $this->marcacao = $marcacao; }` (forma PHP 7.1 sem CPP).

### 3. ApuracaoService (rewrite completo) — CLT completa

`Modules/PontoWr2/Services/ApuracaoService.php` — agora todas as regras CLT implementadas:

- **`aplicarRegraTolerancia($a)`** — Art. 58 §1º. Config: `pontowr2.clt.tolerancia_minutos_por_marcacao` (default 5) e `tolerancia_maxima_diaria_minutos` (default 10).
- **`aplicarRegraIntrajornada($a)`** — Art. 71. Se `realizada_trabalhada_minutos > 360` e `realizada_intrajornada_minutos < 60`, registra violação = `60 - realizada`.
- **`aplicarRegraInterjornada($a, $apuracaoAnterior)`** — Art. 66. Se `gap < 660min` (11h), registra violação.
- **`aplicarRegraHoraExtra($a, $colab, $data)`** — Art. 59. Excedente = `trabalhada - prevista`; limita a 2h/dia; divide HE em diurna/noturna via `dividirDiurnoNoturno()`.
- **`aplicarRegraAdicionalNoturno($a)`** — Art. 73. 22h-5h, +20%, hora reduzida 52'30" (considerada via `fatorHoraNoturnaReduzida = 60/52.5`).
- **`aplicarRegraDsr($a)`** — Lei 605/49 Art. 9º. Repercussão HE sobre repouso semanal proporcional.
- **`aplicarIntercorrencias($a)`** — ATESTADO/CONSULTA abonam; REUNIAO/VISITA contam como trabalhado; ESQUECIMENTO ajusta marcação; OUTRO adiciona divergência.

Utilitários:

- **`dividirDiurnoNoturno(Carbon $inicio, Carbon $fim)`** — retorna `['diurno_minutos' => X, 'noturno_minutos' => Y]`. Janela noturna = [00:00-05:00] ∪ [22:00-24:00] em cada dia do intervalo. Lida com cruzamento de meia-noite percorrendo dia-a-dia e usando `intersecaoMinutos($ini1, $fim1, $ini2, $fim2)`.
- **`addDivergencia($a, $chave, $mensagem)`** — agrega em `$a->divergencias` e marca `$a->estado = DIVERGENCIA`.

Constantes: `NOTURNO_INICIO_H = 22`, `NOTURNO_FIM_H = 5`.

### 4. BancoHorasService::expirarSaldosAntigos — FIFO com idempotência

Implementação:

1. Calcula `$corte = now()->subMonths($prazoMeses)` (default 6, da `config('pontowr2.banco_horas.prazo_expiracao_meses')`).
2. Soma `creditos_antigos` = SUM onde `tipo=CREDITO` e `data_referencia < $corte`.
3. Soma `debitos_historicos` = SUM de todos débitos históricos até `$corte` (já consumiram FIFO os créditos mais antigos).
4. `$residual = creditos_antigos - debitos_historicos`. Se > 0, há saldo vencido a expirar.
5. Marcador idempotente: `"FIFO-" . $corte->format('Ymd')` em `observacao`. Se já existe movimento `tipo=EXPIRACAO` com esse marcador, pula.
6. Caso contrário, `movimentar($colab, -$residual, TIPO_EXPIRACAO, $marcador)`.

Rodar duas vezes no mesmo dia é seguro (marcador garante idempotência).
