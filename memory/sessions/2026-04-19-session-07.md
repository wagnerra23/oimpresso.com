## Estratégia

Para não estourar contexto nem introduzir defeitos por velocidade:

1. Levantar contrato de cada controller (action → view → variáveis) em uma única leitura em paralelo;
2. Para views que compartilham form (create/edit), extrair uma partial `_form.blade.php`;
3. Manter o mesmo vocabulário visual do resto do módulo (content-header → filtros → widget com tabela → paginação);
4. Reusar helpers inline `$fmtMin` (minutos → HH:MM) via closures PHP 5.3+ (nunca arrow fn);
5. Substituir todo null-safe (`?->`) por `optional(...)->` em cada cadeia que toca relações;
6. Grep final por sintaxe proibida (PHP 8+, Laravel 9+) antes de concluir.

## Verificação

Greps em `Modules/PontoWr2/Resources/views/`:

```
\?->|\bfn\s*\(|\bmatch\s*\(|HasFactory|HasUuids|::factory\(\)                     → 0 matches ✅
^\s+[a-z_]+:\s+\$|public function __construct\([^)]*\b(public|protected|private)\s → 0 matches ✅
```

Padrões de compat usados em **todas** as views:

- `optional(X)->` em vez de `?->` (inclusive encadeado `optional(optional($a)->b)->c`)
- Closure explícita `function ($m) { return ...; }` em vez de arrow fn
- `intdiv()` para divisão inteira
- `list(...) = explode(...)` em vez de short-list `[$a, $b] = ...`
- Sem typed properties, sem CPP, sem match(), sem named args
- Sem `[...]` spread; sem `??=`
