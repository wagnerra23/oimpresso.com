### 2) Migração `laravelcollective/html` → `spatie/laravel-html`

Os ~6.433 usos de `Form::` em 460 Blade views migrados via **shim** `App\View\Helpers\Form`:
- Preserva API estática (zero mudança nas views)
- Delega internamente pra spatie/laravel-html
- Divergências forçadas pra paridade laravelcollective:
  - `checked/selected` XHTML-style (`checked="checked"`)
  - `textarea` conteúdo pré-escapado (spatie não escapa, era XSS)
  - `submit` renderiza `<input type="submit">`, não `<button>`
  - `Form::select` com `placeholder` option vira `<option value="">` prepended
  - Aceita `Collection/Arrayable/Traversable` em `Form::select($list)`

### 5) Bug real achado + corrigido: `/products` "Sem dados"

Browser test revelou que `/products` exibia 0 produtos (havia 72 no banco). Diagnóstico via Chrome MCP:
- Server AJAX retornava `recordsTotal:0` devido a filtros default (`brand_id:1, category_id:22, tax_id:1, unit_id:1`)
- Select de filtro exibia primeiro item como `selected` por default
- Causa: spatie `Html::select()` NÃO suportava `placeholder` option — virava `<select placeholder="All">` (atributo HTML inválido) em vez de prepend `<option value="">All</option>`

Fix no shim: `Form::select` extrai `placeholder` do attrs e usa `Html::select()->placeholder()` nativo do spatie.
